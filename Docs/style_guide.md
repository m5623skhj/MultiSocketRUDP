## 1. 문서 언어와 문체

- **본문**: 한국어
- **코드 식별자**: 원문 그대로 유지 (영문). 예: `SendPacket`, `RUDPSession`, `DISCONNECT_REASON`
- **기술 용어**: 한국어 번역 없이 원문 사용. 예: "CWND", "ACK", "RIO", "worker thread"
- **문체**: `~한다` / `~된다` **설명체** 사용. `~합니다` / `~됩니다` 금지

## 2. 용어 통일 규칙

### 동사

| 허용 표현 | 금지 표현 |
|-----------|-----------|
| 반환한다 | 리턴한다, return한다 |
| 호출한다 | call한다 |
| 등록한다 | register한다 |
| 생성한다 | create한다, new한다 |
| 해제한다 | release한다, free한다 (`Free()` 함수명 자체는 허용) |
| 초기화한다 | initialize한다, init한다 |
| 전송한다 | send한다 |
| 수신한다 | receive한다, recv한다 |
| 전이한다 | transition한다 |
| 할당한다 | alloc한다 |

### 명사

| 허용 표현 | 금지 표현 |
|-----------|-----------|
| 파라미터 | 매개변수, 인수, argument |
| 반환값 | 리턴값, return value |
| 전제 조건 | 사전 조건, precondition |
| 한정자 | qualifier, modifier (단, `[[nodiscard]]` 등 코드 자체는 그대로) |
| 출력 파라미터 | out parameter (`OUT` 매크로 코드 자체는 그대로) |
| 세션 | session (한글 본문에서 단독 사용 시) |
| 핸들러 | handler (한글 본문에서 단독 사용 시) |
| 스레드 | thread (한글 본문에서 단독 사용 시) |
| 풀 | pool (한글 본문에서 단독 사용 시) |
| 큐 | queue (한글 본문에서 단독 사용 시) |

### 상태·결과 표현

- 성공 → `true 반환` 또는 `성공` (문맥에 맞게)
- 실패 → `false 반환` / `nullptr 반환` / `실패` (문맥에 맞게)
- 미정의 동작 → `미정의 동작(UB)` 또는 `동작 미보장`
- 해제 진행 중인 상태 → `해제 진행 중 (RELEASING)`으로 일관되게 표현

## 3. 코드 블록 규칙

### 언어 태그

| 코드 종류 | 언어 태그 |
|-----------|-----------|
| C++ 헤더 / 소스 | ` ```cpp ` |
| C# 소스 | ` ```csharp ` |
| 흐름 / 의사코드 / ASCII 다이어그램 | ` ``` ` (태그 없음) |
| 커맨드라인 | ` ```bash ` |
| JSON / YAML | ` ```json ` / ` ```yaml ` |
| Python | ` ```python ` |

### 시그니처 블록 작성 규칙

**`[[nodiscard]]` 한정자** — 코드 블록 안에 별도 줄로 포함한다.

```cpp
[[nodiscard]]
bool IsConnected() const;
```

**`OUT` 매크로** — 출력 파라미터임을 나타내므로 시그니처에 그대로 포함한다.

```cpp
[[nodiscard]]
bool DoSend(OUT RUDPSession& session, ThreadIdType threadId) const;
```

**`noexcept`** — 예외 보장이 있을 때 시그니처에 그대로 포함한다.

```cpp
void OnRetransmissionTimeout() noexcept;
```

**enum 파라미터** — 시그니처에 포함하고, 허용값을 별도 표로 문서화한다.

```cpp
void DoDisconnect(const DISCONNECT_REASON disconnectReason);
```

**삭제된 생성자/대입 연산자** — 클래스 개요 또는 생성자 섹션에서 복사·이동 불가 여부를 명시한다.

```cpp
// 복사·이동 불가
RUDPIOHandler(const RUDPIOHandler&) = delete;
RUDPIOHandler& operator=(const RUDPIOHandler&) = delete;
```

### 사용 예시 블록

- 실제로 컴파일·실행 가능한 코드를 원칙으로 한다.
- `// ✅ 안전`, `// ❌ 위험`, `// ⚠️ 주의` 인라인 주석으로 올바른 패턴과 잘못된 패턴을 대비한다.
- 실전 패턴(브로드캐스트, 에러 처리 등)이 있으면 함께 제시한다.

### 내부 동작 흐름

- 언어 태그 없는 코드 블록(의사코드)으로 표시한다.
- 단계는 번호(`1.`, `2.`)나 `→` 화살표로 표현한다.
- 실제 소스 발췌는 간략화해도 되지만 핵심 분기·반환 조건은 유지한다.
- `├─`, `└─`, `│`, `►`, `──`, `▼` 등 유니코드 박스 문자를 적극 활용한다.

## 4. C++ 코드 주석과 문서의 관계

코드 내 `// ----------------------------------------` 구분선 블록 주석이 문서화의 기준이 된다.

### 주석 태그 → 문서 섹션 매핑

| 코드 주석 태그 | 문서에서의 처리 |
|---------------|----------------|
| `// @brief` | 함수 설명 첫 서술 |
| `// @details` | 함수 설명 추가 서술 또는 **내부 동작** 섹션 |
| `// @param 이름 설명` | **파라미터** 표의 해당 행 설명 |
| `// @return 설명` | **반환값** 섹션 설명 |

**예시 — 코드 주석에서 문서로 변환:**

코드:
```cpp
// ----------------------------------------
// @brief 현재 세션을 RELEASING 상태로 전이시키고 연결 해제 프로세스를 시작합니다.
// @details 예약 상태 또는 연결 상태에서만 RELEASING 상태로 전이할 수 있습니다.
// ----------------------------------------
void DoDisconnect(const DISCONNECT_REASON disconnectReason);
```

문서:
```markdown
### `DoDisconnect`

```cpp
void DoDisconnect(const DISCONNECT_REASON disconnectReason);
```

현재 세션을 RELEASING 상태로 전이시키고 연결 해제 프로세스를 시작한다.
예약(RESERVED) 또는 연결(CONNECTED) 상태에서만 전이 가능하다.

| `disconnectReason` | 의미 |
|--------------------|------|
| `BY_ERROR` | 내부 오류로 인한 강제 해제 |
| `BY_CLIENT_REQUEST` | 클라이언트 요청에 의한 정상 해제 |
```

### `[[nodiscard]]` 함수

반환값 설명 뒤에 반드시 아래 문장을 추가한다.

> 반환값을 무시하면 컴파일 경고가 발생한다. 호출 측에서 반드시 검사해야 한다.

### 클래스 레벨 주석

클래스 선언 위의 `//` 블록 주석은 문서 최상단 `>` 인용구에 반영한다.

```cpp
// RUDPIOHandler 클래스는 RIO(Registered I/O) 기반의 네트워크 통신을 처리하는 핸들러입니다.
// 세션 관리, 패킷 송수신, 재전송 처리 등 RUDP 프로토콜의 핵심 I/O 로직을 담당합니다.
class RUDPIOHandler : public IIOHandler { ... };
```

→ 문서:

```markdown
> **RIO(Registered I/O) 기반 네트워크 통신 핸들러.**  
> 세션 관리, 패킷 송수신, 재전송 처리 등 RUDP 프로토콜의 핵심 I/O 로직을 담당한다.
```

## 5. C# 코드 주석과 문서의 관계

C# 코드는 XML 문서 주석(`///`) 없이 인라인 `//` 주석만 사용한다.  
클래스·메서드 설명은 코드 구조와 구현에서 직접 파악해서 문서화한다.

### C# 특유 패턴 문서화

**Primary Constructor (C# 12+)**: 파라미터를 표로 정리하고, `?`(null 허용) 여부를 명시한다.

```csharp
public class RuntimeContext(Client client, NetBuffer? packet)
```

| 파라미터 | 타입 | 설명 |
|----------|------|------|
| `client` | `Client` | 이 컨텍스트를 소유하는 클라이언트 인스턴스 |
| `packet` | `NetBuffer?` | 트리거된 수신 패킷. 없으면 `null` |

**계산 프로퍼티 (`=>`)**:

```markdown
**`AverageExecutionTimeMs`** — 총 실행 시간 / 실행 횟수. 실행 횟수가 0이면 0 반환.

**`SuccessRate`** — 성공 횟수 / 전체 실행 횟수 × 100 (%). 실행 횟수가 0이면 0 반환.
```

**`internal` 메서드**: 외부 API 문서에서 원칙적으로 생략한다. 동작 이해에 필수적인 경우 **내부 메서드** 섹션을 별도로 두고 간략히 서술한다.

**`Lock` / `ConcurrentDictionary` 등 동시성 타입**: 스레드 안전성 여부와 보호 대상을 **동시성 보호** 섹션 또는 주의사항에 명시한다.

## 6. Python 코드 주석과 문서의 관계

Python 코드는 함수·클래스 docstring (`"""..."""`)과 `#` 인라인 주석을 사용한다.

### Docstring 태그 → 문서 섹션 매핑

| Docstring 섹션 | 문서에서의 처리 |
|----------------|----------------|
| 첫 줄 요약 | 함수 설명 첫 문장 |
| `Args:` | **파라미터** 표 |
| `Returns:` | **반환값** 섹션 |
| `Raises:` | **예외** 섹션 |

**예시 — Python docstring에서 문서로 변환:**

코드:
```python
def extract_function_info(
    changed: ChangedFunction,
    file_content: str | None,
) -> FunctionInfo | None:
    """
    변경된 함수에 대해 main 브랜치의 최종 코드에서 구조화된 정보를 추출한다.

    Args:
        changed: ChangedFunction 객체.
        file_content: 해당 파일의 최종 내용.

    Returns:
        FunctionInfo 객체. DELETED이거나 추출 실패 시 None.
    """
```

문서:
```markdown
### `extract_function_info`

```python
def extract_function_info(
    changed: ChangedFunction,
    file_content: str | None,
) -> FunctionInfo | None:
```

변경된 함수에 대해 main 브랜치의 최종 코드에서 구조화된 정보를 추출한다.

| 파라미터 | 타입 | 설명 |
|----------|------|------|
| `changed` | `ChangedFunction` | 변경된 함수 정보 |
| `file_content` | `str \| None` | 해당 파일의 최종 내용 |

**반환값**: `FunctionInfo` 객체. `DELETED` 변경이거나 추출 실패 시 `None`.
```

## 7. 섹션 구조 컨벤션

### 최상위 문서 구조

```
# 클래스명 또는 모듈명

> **한 줄 역할 요약 (굵게 강조)**  
> 두 번째 줄: 주요 특징 또는 설계 의도  
> 세 번째 줄: 연계 클래스 또는 사용 방법 힌트 (선택)

---

## 목차

1. [섹션 제목](#anchor)
...

---

## 1. 섹션 제목
...

---

## 관련 문서
- [[문서명]] — 한 줄 설명
```

### 함수/메서드 섹션 내부 순서

해당 항목이 없으면 생략한다.

1. **시그니처** — 코드 블록 (`[[nodiscard]]`, `OUT`, `noexcept` 등 포함)
2. **전제 조건** — 호출 전 반드시 성립해야 하는 상태 (있을 때만)
3. **파라미터** — 표 또는 인라인 설명 (enum 허용값 포함)
4. **반환값** — 표 또는 인라인 설명 (`[[nodiscard]]` 경고 포함)
5. **내부 동작** — 흐름 코드 또는 의사코드 (복잡한 함수에만)
6. **사용 예시** — 컴파일 가능한 코드 블록
7. **주의사항** — `>` 인용구 또는 `> **주의:**` 강조 인용구

### 섹션 헤딩 레벨

| 레벨 | 용도 |
|------|------|
| `#` | 문서 제목 (클래스/모듈명) |
| `##` | 주요 섹션 (목차 항목) |
| `###` | 함수/메서드 이름 또는 세부 주제 |
| `####` | 세부 항목 (내부 메서드, 특수 케이스 등) |

## 8. 파라미터·반환값 문서화 규칙

### enum 파라미터

`DISCONNECT_REASON`, `SESSION_STATE`, `SEND_PACKET_INFO_TO_STREAM_RETURN` 등 enum 타입 파라미터는 허용값 전체를 표로 문서화한다.

| `disconnectReason` 값 | 의미 |
|----------------------|------|
| `BY_ERROR` | 내부 오류로 인한 강제 해제 |
| `BY_CLIENT_REQUEST` | 클라이언트 요청에 의한 정상 해제 |

### `OUT` 파라미터

파라미터 표의 설명란에 **[출력]** 을 표시한다.

| 파라미터 | 타입 | 설명 |
|----------|------|------|
| `session` | `RUDPSession&` | **[출력]** 전송 대상 세션 |
| `threadId` | `ThreadIdType` | 호출 스레드 ID |

### bool 반환값

`true`/`false` 각각의 조건을 표로 명시한다.

| 반환값 | 조건 |
|--------|------|
| `true` | RIO Send 작업이 성공적으로 예약됨 |
| `false` | 세션이 CONNECTED 상태가 아니거나, 보류 큐가 가득 참 |

### `[[nodiscard]]` 함수

반환값 설명 뒤에 아래 문장을 추가한다.

> 반환값을 무시하면 컴파일 경고가 발생한다. 호출 측에서 반드시 검사해야 한다.

## 9. 표 사용 기준

표를 사용하는 경우:

- 파라미터가 3개 이상이거나 타입·설명이 모두 필요할 때
- `true`/`false` 이외의 다양한 반환 의미가 있을 때
- enum 허용값을 나열할 때
- 두 가지 이상의 옵션·패턴·설정을 비교할 때
- 이벤트/훅의 호출 스레드·시점·소켓 상태를 정리할 때

표를 사용하지 않는 경우:

- 파라미터가 1~2개이고 설명이 짧을 때 → 인라인 서술
- 셀 내용에 줄바꿈이 많이 필요할 때 → 소제목(`####`) + 서술로 분리

외부 SVG 없이 간단한 흐름을 표현할 때 코드 블록(태그 없음) 안에 ASCII 아트를 사용한다.

- `──`, `►`, `│`, `↓`, `→`, `└`, `├`, `▼` 등 유니코드 문자를 적극 활용한다.
- `✅` / `❌` / `⚠️` 이모지는 주의사항·안전/위험 패턴 표시에만 사용한다.

---

## 11. Obsidian 위키링크

```markdown
[[문서명]]              // 같은 Vault 내 문서 링크
[[문서명|표시 텍스트]]  // 링크 텍스트를 바꿀 때
![[파일명.svg]]         // SVG 이미지 삽입
```

- **기존 위키링크는 절대 제거하거나 일반 마크다운 링크로 바꾸지 않는다.**
- 신규 문서에서도 관련 문서 참조는 위키링크를 사용한다.
- 링크 대상 문서명은 확장자 없이 작성한다 (`.md` 생략).
- 문서 하단 `## 관련 문서` 섹션에 위키링크로 관련 문서를 나열한다.

---

## 12. 신규 문서 생성 규칙

### 파일 위치와 이름

- 파일명은 클래스명과 일치시킨다 (PascalCase).
- `config.py`의 `CODE_TO_DOCS_DIR_MAP` 매핑 경로에 생성한다.
- 클래스명과 파일명이 다를 경우 `CLASS_TO_DOC_OVERRIDE`에 등록한다.

### 최소 포함 항목

1. 문서 제목 (`# ClassName`)
2. 한 줄 역할 요약 (`>` 인용구 — 클래스 주석에서 추출)
3. 목차 (섹션이 3개 이상인 경우)
4. 생성자/초기화 섹션 (항상 최상단)
5. 각 public API 섹션 (시그니처 → 전제 조건 → 파라미터 → 반환값 → 사용 예시 순)
6. 동시성 보호 섹션 (`Lock`, `atomic`, `mutex` 등 사용 시)
7. 관련 문서 섹션

### 복사·이동 불가 클래스

`= delete` 생성자/대입 연산자가 있는 경우 개요에 명시한다.

```markdown
> **복사 및 이동 불가.** 단일 인스턴스로만 사용한다.
```

## 13. 금지 사항

- **인터페이스(시그니처·파라미터·반환 의미)가 변경되지 않은 경우** 문서를 수정하지 않는다.
- **기존 위키링크(`[[...]]`, `![[...]]`)를 제거하거나 일반 링크로 변환하지 않는다.**
- **`~합니다` / `~됩니다` 문체를 사용하지 않는다.**
- **영문 동사를 그대로 한글화하지 않는다.** (`return한다` → `반환한다`)
- **`[[nodiscard]]` 한정자를 시그니처에서 생략하지 않는다.**
- **`OUT` 매크로를 파라미터 표에서 누락하지 않는다.**
- **enum 파라미터의 허용값 표를 누락하지 않는다.**
- **코드 블록 언어 태그를 누락하지 않는다.** (C++ → `cpp`, C# → `csharp`, Python → `python`)
- **목차 없이 섹션이 5개 이상인 문서를 작성하지 않는다.**
- **불필요한 굵게·이탤릭 강조를 남발하지 않는다.** 핵심 경고·용어에만 사용한다.
